vect2 C++ Lvl 1

A 2D integer vector. Pure operator overloading practice — the gentlest exercise, do it first to warm up.

🎯 The concept

Store two ints x, y. You must support +, -, scalar * (both v*2 and 2*v), [] indexing (read & write), comparisons, and printing as {x, y}.

  • Why two operator[]? A non-const one returns int& (so v[1]=12 works) and a const one returns int (so a const vect2 can still be read).
  • Why friend for 2 * v? The left operand is an int, not a vect2, so it can't be a member — make it a free friend function.
  • << is always a friend free function returning std::ostream& so you can chain cout << v << endl.

🧩 Memory hooks

  • Pre vs post ++: pre returns *this (a ref); post takes a dummy int, saves tmp, mutates, returns tmp (by value).
  • Compound (+=) returns vect2&; binary (+) returns vect2 by value and is const.
  • No heap, no destructor needed — but still write the canonical 4 (default, copy, assign) for habit.

⚠️ Traps

  • Forgetting const on binary ops & operator[] const → const objects won't compile.
  • Returning vect2 instead of vect2& from += breaks v2 += v2 += v3 chaining.
  • Print must be exactly {x, y} (note the space after comma).
vect2.hpp — the signatures to memorize
class vect2 {
  int x, y;
public:
  vect2();
  vect2(int x, int y);
  vect2(const vect2& o);
  vect2& operator=(const vect2& o);

  int& operator[](int i);          // write
  int  operator[](int i) const;    // read

  vect2& operator++();              // pre
  vect2  operator++(int);           // post
  vect2& operator+=(const vect2& v);
  vect2  operator+(const vect2& v) const;
  vect2  operator*(int s) const;
  friend vect2 operator*(int s, const vect2& v);
  bool operator==(const vect2& v) const;
  friend std::ostream& operator<<(std::ostream&, const vect2&);
};
vect2.cpp — the patterns
// post-increment: save, mutate, return old
vect2 vect2::operator++(int) {
  vect2 tmp = *this;
  x++; y++;
  return tmp;
}
// compound returns ref for chaining
vect2& vect2::operator+=(const vect2& v) {
  x += v.x; y += v.y; return *this;
}
// 2 * v  -> free friend function
vect2 operator*(int s, const vect2& v) {
  return vect2(v[0] * s, v[1] * s);
}
std::ostream& operator<<(std::ostream& os, const vect2& v) {
  os << "{" << v[0] << ", " << v[1] << "}";
  return os;
}

bigint C++ Lvl 1

Arbitrary-precision unsigned integer. Store digits in a std::vector<int> — easiest to manage.

🎯 The concept

Store digits least-significant first (reversed). This makes addition & left-shift trivial. Required: build from unsigned long long and std::string, addition, comparisons, and digit-shift (<< multiplies by 10ⁿ, >> divides). Print with no leading zeros.

  • Why store reversed? digits[0] = units. Adding aligns naturally; shifting left = insert zeros at front; printing iterates back-to-front.
  • trim() removes trailing zeros (which are leading zeros in real order) but keeps at least one digit.

🧩 Memory hooks

  • Addition: loop while (i < size || carry), sum = carry (+ digits…), push sum%10, carry = sum/10.
  • << n = insert(begin(), n, 0) (but return 0 if value is 0). >> n = erase first n, then trim.
  • Comparison: after trim, longer vector = bigger; if equal length, compare digit-by-digit from the most-significant end (back to front).
  • Provide both int and bigint overloads of the shifts — the main uses (d >>= (const bigint)2).

⚠️ Traps

  • Leading zeros in output → always trim() after constructing/adding.
  • The bigint-arg shifts need a private to_int() helper to convert.
  • == compares sizes first; build everything else (!= <= >=) on top of <, >, ==.
  • Don't forget the full canonical form incl. destructor (vector handles memory, but write ~bigint(){}).
bigint.cpp — addition (the core)
bigint bigint::operator+(const bigint& b) const {
  bigint res; res.digits.clear();
  int carry = 0;
  size_t n = std::max(digits.size(), b.digits.size());
  for (size_t i = 0; i < n || carry; i++) {
    int sum = carry;
    if (i < digits.size())   sum += digits[i];
    if (i < b.digits.size()) sum += b.digits[i];
    res.digits.push_back(sum % 10);
    carry = sum / 10;
  }
  res.trim();
  return res;
}
bigint.cpp — shifts, compare, print
// << shifts digits left (multiply by 10^shift)
bigint bigint::operator<<(int shift) const {
  if (*this == bigint(0)) return bigint(0);
  bigint res = *this;
  res.digits.insert(res.digits.begin(), shift, 0);
  return res;
}
bool bigint::operator<(const bigint& b) const {
  if (digits.size() != b.digits.size())
    return digits.size() < b.digits.size();
  for (int i = digits.size()-1; i >= 0; i--)
    if (digits[i] != b.digits[i])
      return digits[i] < b.digits[i];
  return false;
}
std::ostream& operator<<(std::ostream& os, const bigint& b) {
  for (int i = b.digits.size()-1; i >= 0; i--)
    os << b.digits[i];
  return os;
}

polyset C++ Lvl 1

The hardest Lvl 1. Pure polymorphism + inheritance + canonical form. They give you bag, searchable_bag, array_bag, tree_bag. You write the searchable subclasses + a set wrapper.

🎯 The concept

  • bag = abstract base (insert / print / clear). searchable_bag = abstract, adds has(int). Both use virtual public inheritance from bag → this is the diamond setup.
  • Part 1: searchable_array_bag inherits array_bag + searchable_bag and implements has(). Same for the tree version (BST search!).
  • Part 2: set wraps a searchable_bag& reference and only inserts if !has(value) → guarantees uniqueness.
The diamond: Because array_bag and searchable_bag both inherit bag, the virtual public bag keyword on each is what prevents two copies of bag. Memorize the inheritance lines exactly.

🧩 Memory hooks

  • Searchable subclass canonical form: the copy ctor forwards to the base: : array_bag(copy); assignment calls array_bag::operator=(copy) after the self-check.
  • array has(): linear loop over data[i]. tree has(): BST walk — go left if value < node, right otherwise.
  • set stores a reference searchable_bag& bag; → no copy assignment possible (a reference can't be reseated) → leave operator= out / commented.

⚠️ Traps

  • Forgetting virtual public bag → diamond ambiguity, won't compile.
  • Forgetting const on has() and print() → the main calls them on a const object.
  • In set, the body of insert must check membership before inserting — that's the whole point.
  • The protected members (data/size, tree) are accessible from your subclass — use them directly in has().
searchable_array_bag.* — inheritance + has()
class searchable_array_bag
  : public array_bag, public searchable_bag {
public:
  searchable_array_bag();
  searchable_array_bag(const searchable_array_bag& c);
  searchable_array_bag& operator=(const searchable_array_bag& c);
  ~searchable_array_bag();
  bool has(int value) const;
};
// .cpp
searchable_array_bag(const searchable_array_bag& c)
  : array_bag(c) {}
searchable_array_bag& operator=(const searchable_array_bag& c) {
  if (this != &c) array_bag::operator=(c);
  return *this;
}
bool has(int value) const {
  for (int i = 0; i < size; i++)
    if (data[i] == value) return true;
  return false;
}
searchable_tree_bag has() + set wrapper
bool searchable_tree_bag::has(int value) const {
  node* cur = tree;
  while (cur) {
    if (cur->value == value) return true;
    else if (cur->value > value) cur = cur->l;
    else cur = cur->r;
  }
  return false;
}
// set.hpp: stores a REFERENCE
class set {
  searchable_bag& bag;
public:
  set(searchable_bag& b) : bag(b) {}
  void insert(int v) { if (!bag.has(v)) bag.insert(v); }
  bool has(int v) const { return bag.has(v); }
};

bsq C Lvl 2

Biggest Square. File parsing + validation + dynamic programming. The DP is short; most of your time is correct parsing & validation.

🎯 The concept

  • First line = [number of rows][empty char][obstacle char][full char]. Read it, validate it. Last 3 chars are the three symbols (must be distinct); everything before is the row count (must be all digits, > 0).
  • Map lines: all must be the same length, end in \n, contain only empty/obstacle chars. Reject extra trailing lines.
  • DP recurrence: for each non-obstacle cell, dp[i][j] = 1 + min(up, left, diag); obstacle → 0; first row/col → 1.
  • Track the max & its bottom-right corner. Ties → keep the topmost then leftmost square. Then fill that square with the full char and print.
Any failure in validation → print exactly Error: invalid map\n to stdout. Allowed funcs only: malloc, calloc, realloc, free, fopen, fclose, getline, fscanf, fputs, fprintf, printf.

🧩 Memory hooks

  • min3(up,left,diag) is the whole algorithm. Flatten the dp to a 1D array dp[i*C + j] to avoid VLA hassles.
  • Tie-break: compute each square's top-left as corner - size + 1 and compare (top first, then left).
  • Read first line with getline, strip the trailing \n, then peel the 3 symbol chars off the end.
  • No args → read stdin; one arg → fopen it.

⚠️ Traps

  • Lines of differing length = invalid. Track cols from first map line, compare every line.
  • Three symbols must be pairwise distinct; first line must have ≥1 digit before them.
  • An empty map / 0 rows = invalid. A char not in the alphabet = invalid.
  • Don't forget to free everything (the map rows, the dp, the getline buffer).
bsq.c — the DP core
for (int i = 0; i < R; i++)
 for (int j = 0; j < C; j++) {
   int idx = i * C + j;
   if (map[i][j] == obst) dp[idx] = 0;
   else if (i == 0 || j == 0) dp[idx] = 1;
   else {
     int up   = dp[(i-1)*C + j];
     int left = dp[i*C + (j-1)];
     int diag = dp[(i-1)*C + (j-1)];
     dp[idx] = 1 + min3(up, left, diag);
   }
   if (dp[idx] > best) { best = dp[idx]; bi = i; bj = j; }
 }
// fill: top = bi-best+1, left = bj-best+1 -> set full_ch

life C Lvl 2

Conway's Game of Life with a pen-drawing input phase. Two sub-problems: parse pen commands, then simulate N generations.

🎯 The concept

  • Args: ./life width height iterations. A pen starts top-left. Commands: w/a/s/d move (clamped to board), x toggles drawing. While drawing, mark the current cell alive.
  • After EOF, run the Game of Life for iterations generations and print: 'O' = alive, space = dead, newline per row.
  • Rules: live cell with 2 or 3 neighbors survives; dead cell with exactly 3 neighbors is born; else dead.
Allowed funcs: atoi, read, putchar, malloc, calloc, realloc, free. So input is read(0, &c, 1) char-by-char, output is putchar.

🧩 Memory hooks

  • Double buffer: use grid[2][...][...] and ping-pong with cur = it%2, next = (it+1)%2 so neighbor counts use the frozen previous generation.
  • Padded border: size the grid (height+2) x (width+2) and work on the inner 1..height / 1..width so out-of-bounds neighbors are just the dead padding — no edge checks needed.
  • Neighbor count: double loop dy,dx ∈ {-1,0,1}, skip (0,0).
  • Final buffer to print is iterations % 2.

⚠️ Traps

  • Pen must not leave the board (clamp every move). Default pen is up (not drawing).
  • Mark the cell alive only while the pen is down — including the cell you start on after x.
  • Don't add neighbor checks for borders if you use padding — that's the whole trick.
  • Validate ac == 4 and non-negative dimensions.
life.c — input + simulation
// padded grid: indices 1..height, 1..width
int grid[2][height+2][width+2] = {0};
int x = 1, y = 1, pen = 0; char c;
while (read(0, &c, 1) > 0) {
  if (c=='w' && y>1) y--;
  else if (c=='s' && y<height) y++;
  else if (c=='a' && x>1) x--;
  else if (c=='d' && x<width) x++;
  else if (c=='x') pen = !pen;
  if (pen) grid[0][y][x] = 1;
}
for (int it=0; it<iterations; it++) {
  int cur=it%2, next=(it+1)%2;
  for (int i=1; i<=height; i++)
   for (int j=1; j<=width; j++) {
     int n=0;
     for (int dy=-1; dy<=1; dy++)
      for (int dx=-1; dx<=1; dx++)
        if (dx||dy) n += grid[cur][i+dy][j+dx];
     grid[next][i][j] = grid[cur][i][j]
       ? (n==2 || n==3) : (n==3);
   }
}

📋 Cheat Sheet

A dense, one-page reference of every key signature, pattern, and rule.

vect2 C++

  • Members int x, y; + canonical 4 (default, copy, operator=, dtor)
  • int& operator[](int) write · int operator[](int) const read
  • Pre vect2& operator++() · Post vect2 operator++(int) → save tmp, return old
  • += returns vect2& · + returns vect2 and is const
  • v*2 = member · 2*v = friend operator*(int, const vect2&)
  • friend ostream& operator<< prints exactly {x, y} (space after comma)

bigint C++

  • vector<int> digits, least-significant first
  • Ctors from unsigned long long & std::string
  • Add: while (i < n || carry) → push sum%10, carry = sum/10
  • <<n = insert(begin(), n, 0) · >>n = erase n front + trim()
  • Compare: size first; equal → digit-by-digit MSB→LSB
  • trim() keeps ≥1 digit · print back-to-front, no leading zeros

polyset C++

  • bag abstract → searchable_bag : virtual public bag adds has()
  • searchable_array_bag : public array_bag, public searchable_bag
  • Copy ctor forwards: : array_bag(c) {} · operator= calls array_bag::operator=(c)
  • array has(): linear loop · tree has(): BST walk (l if <, r else)
  • has() & print() must be const
  • set holds searchable_bag& bag; · insert iff !bag.has(v)

bsq C

  • First line: rows + empty + obstacle + full chars (distinct)
  • All map lines equal length, end in \n, only legal chars
  • Any failure → print Error: invalid map\n
  • DP: dp = 1 + min(up, left, diag); obstacle→0; row/col 0→1
  • Tie-break: topmost square, then leftmost
  • Allowed: malloc, calloc, realloc, free, fopen, fclose, getline, fscanf, fputs, fprintf, printf

life C

  • ./life width height iterations · validate ac==4
  • Pen top-left, default UP; wasd move (clamp), x toggle
  • Mark cell alive while pen is down
  • Padded grid (h+2)×(w+2) · double-buffer cur=it%2
  • Rules: live 2–3 survive · dead exactly 3 born · else dead
  • Allowed: atoi, read, putchar, malloc, calloc, realloc, free

⚡ General reflexes

  • Compile early: c++ -Wall -Wextra -Werror *.cpp
  • Always write the full canonical form (Coplien) in C++
  • const-correctness: the provided main calls methods on const objects
  • Match output format byte-for-byte (spaces, newlines)
  • C exercises: free every allocation, no leaks
