Architecture — 3 Layers
Layer 1 (given): array_bag and tree_bag — already implemented, just inherit
Layer 2 (you write): searchable_array_bag and searchable_tree_bag — inherit from both bag + searchable_bag
Layer 3 (you write): set — wraps a searchable_bag*, enforces no-duplicate invariant via contains()
Core Patterns
searchable_bag interface
Pure virtual: virtual bool contains(const T&) const = 0;
Every concrete class must implement contains — check before insert in set::add
set::add — the only logic that matters
if (!bag->contains(val)) bag->add(val); — that's the entire set semantic
set holds a pointer to searchable_bag — polymorphism does the rest
OCF — apply to EVERY class you write
Default constructor (may call base ctor via initializer list)
Copy constructor: deep-copy all heap-allocated members
Copy assignment: self-guard + deep-copy + return *this
Destructor: delete all raw pointers — never leave leaks
Traps
Forgetting OCF on any class — each one needs all 4. Not just the set.
Slicing: set must store a searchable_bag* pointer, not a value — polymorphism requires it.
Missing virtual destructor on base classes — UB when deleting through base pointer.
Multiple inheritance ambiguity — use virtual inheritance if needed on searchable_bag.
Skeleton — set class
template <typename T> class set { private: searchable_bag<T>* _bag; public: set(searchable_bag<T>* b) : _bag(b) {} set(const set &o) : _bag(o._bag->clone()) {} // deep copy via virtual clone() set &operator=(const set &o) { if (this != &o) { delete _bag; _bag = o._bag->clone(); } return *this; } ~set() { delete _bag; } void add(const T &val) { if (!_bag->contains(val)) // set semantic: no duplicates _bag->add(val); } bool contains(const T &val) const { return _bag->contains(val); } int size() const { return _bag->size(); } };
Core Design
Store digits as std::string (easiest) or std::vector<int>
String approach: LSB at index 0 (reversed) or MSB at 0 — pick one, stay consistent
Separate sign: bool _negative; std::string _digits;
Key Algorithms
Addition (same sign)
Walk both strings right-to-left, add digit by digit + carry
Push remainder, carry final overflow
Reverse at end if storing LSB-first
Subtraction
Compare magnitudes first — larger minus smaller
Walk right-to-left, subtract + borrow
Strip leading zeros from result
Traps
Leading zeros in result — always strip with while (size > 1 && front == '0') remove front.
Zero has no sign — make sure -0 normalizes to 0.
Comparison operators must handle sign and then magnitude separately.
OCF still required — if you use raw pointers for digit storage, deep copy in copy ctor/assignment.
Use std::string for digits — it gives you size(), substr(), and easy reversal for free.

Algorithm — one formula
For every non-obstacle cell: dp[i][j] = 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1])
Track the max value and its position as you fill the DP table
Then fill a square of size max starting at that position with the full character
DP Example
input00100
dp →11011

12013
0 = obstacle, highlighted = growing square sizes
Parsing the Map
First line of the map encodes: size obstacle_char empty_char full_char
Read with getline or fscanf — validate before touching the grid
If invalid input → write error to stderr and exit
Traps
Off-by-one in square fill: square starts at (i - size + 1, j - size + 1), not at (i, j).
First row/col: dp[0][j] and dp[i][0] are just 0 or 1 — no min of three needed.
Map validation: wrong number of columns, invalid chars → error output, non-zero exit.
Output: print the modified map (not just the square) to stdout.
Skeleton
// 1. Parse header: obstacle, empty, full chars + expected width // 2. Read map into vector<string>, validate each row // 3. Build DP grid int best = 0, bi = 0, bj = 0; for (int i = 0; i < rows; i++) { for (int j = 0; j < cols; j++) { if (map[i][j] == obstacle) { dp[i][j] = 0; continue; } if (i == 0 || j == 0) dp[i][j] = 1; else dp[i][j] = 1 + std::min({dp[i-1][j], dp[i][j-1], dp[i-1][j-1]}); if (dp[i][j] > best) { best = dp[i][j]; bi = i; bj = j; } } } // 4. Fill square — top-left corner is (bi - best + 1, bj - best + 1) for (int i = bi - best + 1; i <= bi; i++) for (int j = bj - best + 1; j <= bj; j++) map[i][j] = full; // 5. Print map for (auto &row : map) std::cout << row << '\n';
Rules (Conway)
Live cell with 2 or 3 live neighbors → survives
Dead cell with exactly 3 live neighbors → becomes alive
All other cells → die or stay dead
CRITICAL: compute next generation from a copy of current grid — never in-place
Traps
Updating in-place: a cell you already updated affects the count of its neighbor → wrong output.
Boundary: check i >= 0 && i < rows && j >= 0 && j < cols for all 8 neighbors.
Read the subject — number of generations, input format, and output format vary per version.
Off-by-one in neighbor count: loop offset is -1..+1 but skip (0,0) — that's the cell itself.
Skeleton
int countNeighbors(const std::vector<std::string> &grid, int r, int c) { int count = 0, rows = grid.size(), cols = grid[0].size(); for (int dr = -1; dr <= 1; dr++) for (int dc = -1; dc <= 1; dc++) { if (dr == 0 && dc == 0) continue; // skip self int nr = r + dr, nc = c + dc; if (nr >= 0 && nr < rows && nc >= 0 && nc < cols && grid[nr][nc] == '*') count++; } return count; } std::vector<std::string> nextGen(const std::vector<std::string> &grid) { std::vector<std::string> next = grid; // copy — never modify original for (int r = 0; r < (int)grid.size(); r++) for (int c = 0; c < (int)grid[0].size(); c++) { int n = countNeighbors(grid, r, c); if (grid[r][c] == '*') next[r][c] = (n == 2 || n == 3) ? '*' : '.'; else next[r][c] = (n == 3) ? '*' : '.'; } return next; } // main loop: repeat nextGen() for N generations, print final grid
